# VT Relationships 批次查詢使用說明

`fetch_vt_relationships.py` — 批次從 VirusTotal API 查詢 APT IoC 的關聯性資料。

---

## 資料規模（截至 2026-03-16）

| 類型 | 唯一數量 | API 呼叫次數 |
|---|---|---|
| SHA256 (file) | 5,345 | × 4 = 21,380 |
| IP | 2,002 | × 1 = 2,002 |
| Domain | 4,660 | × 1 = 4,660 |
| **總計** | **12,007** | **28,042** |

> 每個 file 查 4 個 endpoint：`contacted_ips`、`contacted_domains`、`dropped_files`、`execution_parents`
> IP / Domain 各查 1 個 endpoint：`resolutions`

---

## 快速開始

### 正常執行（前台，佔用終端）
```bash
cd C:\Users\Win\Documents\thesis
uv run python fetch_vt_relationships.py
```

### 背景執行（不佔終端）
```bash
cd C:\Users\Win\Documents\thesis
nohup uv run python fetch_vt_relationships.py >> vt_relationships/progress.log 2>&1 &
echo "PID: $!"
```
> 記下 PID，需要停止時執行 `kill <PID>`

---

## 斷點續跑

**每次重跑都會自動從中斷處繼續，不需要任何額外設定。**

腳本在寫入結果前會先檢查：
```
vt_relationships/.cache/files/{sha256}.json   → 存在就跳過
vt_relationships/.cache/ips/{ip}.json         → 存在就跳過
vt_relationships/.cache/domains/{domain}.json → 存在就跳過
```

不論是正常結束、手動 kill、或機器關機，明天重新執行同一條指令即可繼續。

## 常用指令

### 先估算，不實際查詢
```bash
uv run python fetch_vt_relationships.py --dry-run
```

### 只跑特定 org
```bash
uv run python fetch_vt_relationships.py --orgs APT28,APT29
uv run python fetch_vt_relationships.py --orgs APT-C-23
```

### 只跑特定 IoC 類型
```bash
# 只查 file（最消耗配額，每個 4 次）
uv run python fetch_vt_relationships.py --ioc-types file

# 只查 domain 和 IP（省配額）
uv run python fetch_vt_relationships.py --ioc-types domain,ip
```

### 調整每日 API 上限
```bash
# 保守（避免 429，每 10 分鐘配額耗盡才 wait）
uv run python fetch_vt_relationships.py --daily-limit 2000

# 預設（Academic plan 上限）
uv run python fetch_vt_relationships.py --daily-limit 5700
```

### 組合使用
```bash
# 背景執行，只跑 file 類型，每日限 3000 次
nohup uv run python fetch_vt_relationships.py \
  --ioc-types file \
  --daily-limit 3000 \
  >> vt_relationships/progress.log 2>&1 &
```

## 429 Rate Limit 說明

VT Academic Plan 限制：**4 req/min，每日上限約 5,700**

腳本遇到 429 時的等待策略：

| 第幾次 429 | 等待時間 |
|---|---|
| 第 1 次 | 60 秒 |
| 第 2 次 | 120 秒 |
| 第 3 次 | 300 秒（5 分鐘）|
| 第 4 次以上 | **600 秒（10 分鐘）** |

每日配額耗盡時，腳本會自動計算等到隔天 UTC 00:00 重置，**無需手動介入**。

## 輸出目錄結構

```
vt_relationships/
├── .cache/                        ← 全局去重快取（跨 org 共用）
│   ├── files/{sha256}.json
│   ├── ips/{ip}.json
│   └── domains/{domain}.json
│
├── {org}/                         ← 各 org 的結果（從 cache 複製）
│   ├── files/{sha256}.json
│   ├── ips/{ip}.json
│   └── domains/{domain}.json
│
├── fetch_stats.json               ← 執行統計（自動更新）
└── progress.log                   ← 詳細執行 log
```

### file 結果格式
```json
{
  "sha256": "abc123...",
  "query_time": "2026-03-16T...",
  "contacted_ips":     [{"id": "1.2.3.4",  "attributes": {...}}, ...],
  "contacted_domains": [{"id": "evil.com", "attributes": {...}}, ...],
  "dropped_files":     [{"id": "def456...", "attributes": {...}}, ...],
  "execution_parents": [{"id": "ghi789...", "attributes": {...}}, ...],
  "errors": []
}
```

### domain / IP 結果格式
```json
{
  "domain": "evil.com",
  "query_time": "2026-03-16T...",
  "resolutions": [
    {"ip_address": "1.2.3.4", "date": 1700000000},
    ...
  ]
}
```

## API Key 設定

腳本依序從以下位置讀取 API Key：

1. `--api-key` 參數
2. 環境變數 `VT_API_KEY`
3. `virustotal_apiTest/.env` 檔案
4. 專案根目錄 `.env` 檔案

`.env` 格式：
```
VT_API_KEY=你的API金鑰
```

## 完整參數說明

```
--vt-results-dir   VT 原始掃描結果目錄（預設：./VT_results）
--output-dir       關聯性結果輸出目錄（預設：./vt_relationships）
--api-key          VT API Key
--orgs             限定查詢的 org（逗號分隔，預設：全部）
--ioc-types        限定 IoC 類型：file,ip,domain（預設：全部）
--daily-limit      每日 API 呼叫上限（預設：5700）
--dry-run          只估算，不實際查詢
```
