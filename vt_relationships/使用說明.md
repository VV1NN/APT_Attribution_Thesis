# VT Relationships 批次查詢使用說明

`scripts/fetch_vt_relationships.py` — 批次從 VirusTotal API 查詢 APT IoC 的關聯性資料。

---

## 查詢的 Relationship Endpoints

### File（每個 SHA256 查 6 個 endpoint）

| Endpoint | 說明 | 覆蓋率 |
|----------|------|--------|
| `contacted_ips` | 檔案執行時連線的 IP | 63.3% |
| `contacted_domains` | 檔案執行時連線的 Domain | 57.8% |
| `contacted_urls` | 檔案執行時連線的 URL | 32.2% |
| `dropped_files` | 檔案釋放/產生的子檔案 | 64.7% |
| `execution_parents` | 產生此檔案的父程序 | 29.7% |
| `bundled_files` | 封裝內含的檔案 | 78.4% |

> 96.4% 的檔案至少有一種 relationship

### Domain（每個 domain 查 6 個 endpoint）

| Endpoint | 說明 | 覆蓋率 |
|----------|------|--------|
| `resolutions` | DNS 解析歷史（domain → IP） | 91.7% |
| `communicating_files` | 與此 domain 通訊的檔案 | — |
| `referrer_files` | 內含此 domain 的檔案 | — |
| `subdomains` | 子網域 | — |
| `historical_ssl_certificates` | 歷史 SSL 憑證 | — |
| `historical_whois` | 歷史 WHOIS 記錄 | — |

### IP（每個 IP 查 5 個 endpoint）

| Endpoint | 說明 | 覆蓋率 |
|----------|------|--------|
| `resolutions` | DNS 反查歷史（IP → domain） | 84.6% |
| `communicating_files` | 與此 IP 通訊的檔案 | — |
| `referrer_files` | 內含此 IP 的檔案 | — |
| `historical_ssl_certificates` | 歷史 SSL 憑證 | — |
| `historical_whois` | 歷史 WHOIS 記錄 | — |

> Enterprise-only endpoints（academic plan 無法使用）：`embedded_domains`, `embedded_ips`, `embedded_urls`, `itw_urls`, `itw_domains`, `itw_ips`, `compressed_parents`, `downloaded_files`

## 快速開始

### 正常執行
```bash
uv run python scripts/fetch_vt_relationships.py
```

### 指定組織與每日上限
```bash
uv run python scripts/fetch_vt_relationships.py \
  --orgs Lazarus_Group,Sandworm_Team,Turla \
  --daily-limit 20000
```

### 背景執行
```bash
nohup uv run python scripts/fetch_vt_relationships.py \
  --daily-limit 20000 \
  >> vt_relationships/progress.log 2>&1 &
echo "PID: $!"
```

### 先估算不實際查詢
```bash
uv run python scripts/fetch_vt_relationships.py --dry-run
```

## 斷點續跑

**每次重跑都會自動從中斷處繼續，不需要任何額外設定。**

腳本在寫入結果前會先檢查全局快取：
```
vt_relationships/.cache/files/{sha256}.json   → 存在就跳過
vt_relationships/.cache/ips/{ip}.json         → 存在就跳過
vt_relationships/.cache/domains/{domain}.json → 存在就跳過
```

不論是正常結束、手動 kill、到達每日限額停止、或機器關機，重新執行同一條指令即可繼續。

## 每日限額控制

| 參數 | 說明 |
|------|------|
| `--daily-limit 20000` | 設定每日 API 呼叫上限（Academic plan: 20,000/day） |
| `--stop-on-quota`（預設） | 到達限額時儲存進度並退出 |
| `--no-stop-on-quota` | 到達限額時 sleep 等待隔天重置 |

## API Quota 與 Rate Limit 說明

### 帳號配額（Academic Plan — `academic_march` group）

| 項目 | 限制 |
|------|------|
| Access level | Limited（standard free public API） |
| Request rate | 20,000 lookups / min |
| Daily quota | 20,000 lookups / day |
| Monthly quota | 620,000 lookups / month |

> **注意事項：**
> - 此為 group 共享額度，同群組內所有使用者的呼叫合計計算
> - 若單一使用者或全部使用者合計耗盡額度，需等到下一個時間窗口重置
> - 最嚴格的限制優先生效（daily 與 monthly 取較低者）
> - 使用限制：不得用於商業工作流程、商業產品或服務

### 腳本內建限速

腳本內建 rate limit：每次請求間隔至少 0.1 秒（`requests_per_min = 600`），遠低於 API 上限以確保穩定性。

### 遇到 429 時的指數退避策略

| 第幾次 429 | 等待時間 |
|------------|----------|
| 第 1 次 | 60 秒 |
| 第 2 次 | 120 秒 |
| 第 3 次 | 300 秒（5 分鐘）|
| 第 4 次以上 | 600 秒（10 分鐘）|

## API Key 設定

腳本依序從以下位置讀取 API Key：

1. `--api-key` 參數
2. 環境變數 `VT_API_KEY`
3. 專案根目錄 `.env` 檔案

`.env` 格式：
```
VT_API_KEY=你的API金鑰
```

## 完整參數說明

```
--vt-results-dir   VT 原始掃描結果目錄（預設：./VT_results）
--output-dir       關聯性結果輸出目錄（預設：./vt_relationships）
--api-key          VT API Key
--orgs             限定查詢的 org（逗號分隔，預設：全部）
--ioc-types        限定 IoC 類型：file,ip,domain（預設：全部）
--daily-limit      每日 API 呼叫上限（預設：500）
--stop-on-quota    到達每日限額時儲存進度並退出（預設開啟）
--no-stop-on-quota 到達每日限額時 sleep 等待隔天
--dry-run          只估算，不實際查詢
```
