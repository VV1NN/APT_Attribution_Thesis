# 概述

用於請 LLM 工具 (例如: `AI Studio`) 泛讀、整理、思考文獻的使用方式

# 提示詞 (Prompt)

## 系統指示

設定 LLM 天生的指示

```text
你是一名網路安全與威脅情報專家，專精於進階持續性威脅 (APT) 分析與攻擊歸因 (Attribution)。
當我提供論文給你時，請你先快速閱讀並整理出你對這篇論文的核心理解。
你同時也是我的論文導讀師。在閱讀過程中，如果我有任何疑問，請你協助我理解論文內容，解答我的問題，並主動補充相關概念。
如果論文中未提及某些關鍵知識，請你結合過往知識和網路搜尋結果來幫助我補充。若我對某些概念感到困惑，請嘗試使用更淺顯易懂的說法或具體的案例（例如著名的 APT 攻擊事件）來為我解釋。
```

## 快速理解一篇文獻

依序使用以下 Prompt

```text
請閱讀這篇論文，並整理出你對這篇論文核心內容的理解，以便我們接下來進行討論。
```

```text
請問這篇論文主要想解決 APT 歸因領域的哪個具體問題或挑戰？其背景為何？
```

```text
這篇論文研究的數據來源 (Data Sources) 是什麼？(例如：IoC、CTI 報告、惡意程式樣本、網路流量等)
```

```text
請為我詳細總結這篇論文的 Introduction 章節，特別是作者提出的研究動機與主要貢獻。
```

```
請詳細介紹這篇論文提出的歸因方法或模型架構。請說明每個分析階段或模組的具體作用，以及每個環節的輸入 (Input) 和輸出 (Output) 分別是什麼？
```

## 文獻重點整理範例

請 LLM 列出我關心的重點整理提示詞

```text
請為我整理出此論文的以下重點：
1. 該論文進行 APT 歸因時，主要依賴哪些特徵或指標 (Features / Indicators)？（例如：TTPs、基礎設施特徵、惡意程式碼特徵等）
2. 作者指出過去的 APT 歸因研究或方法存在哪些缺陷或限制？這篇論文如何解決這些問題？
3. 論文提出的核心歸因演算法或技術是什麼？（例如：機器學習模型、圖神經網路、知識圖譜等）為什麼這個方法能有效提升歸因的準確性或效率？
4. 論文使用了哪些指標來評估其歸因方法的效能？（例如：準確率 (Accuracy)、精確率 (Precision)、召回率 (Recall)、F1-score 等）
5. 論文在評估階段，將其提出的方法與哪些現有的歸因技術或模型進行了比較？結果如何？
```
